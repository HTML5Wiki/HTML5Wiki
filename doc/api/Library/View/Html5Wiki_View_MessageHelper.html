Class Html5Wiki_View_MessageHelper

Maintains a central point where messages can be added during processing of a request.
The main template (template/layout.php) will gather and show them afterwards in the UI using the MessageController.js.

addButton

void addButton( string $text , [bool $showAsButton = true] , [string $javascriptCallback = null] )

After calling an #append*Message-method, use this method to an action to the last appended message.
Actions will usually be displayed as a button or link in the frontend.
You don't have to add an action if you dont want to do anything special.

The UI's library will add a default-close-button to messages, which have no actions assigned (except if autohide was passed as true).
Use $javascriptCallback to inject frontend javascript code for the click-event of your action.

Parameters:

Name Type Description
$text string
$showAsButton bool (optional) Show action as link or as button?
$javascriptCallback string (optional), if not passed, action closes message
